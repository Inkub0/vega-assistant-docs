Reminders

Set up feeding and medication schedules that work even when you're offline.

How reminders work

VEGA schedules reminders directly on your device using the operating system's notification system. They fire at the exact time you set, without needing an internet connection or our servers to be available.

Creating a reminder

  1. Open a pet's profile and tap Schedules.
  2. Tap Add schedule.
  3. Give the schedule a title (e.g. "Morning insulin", "Dinner time").
  4. Optionally add a description for more detail.
  5. Set the recurrence — daily at a fixed time is the most common choice. More advanced schedules use a cron-style expression for full flexibility.
  6. Tap Save. The reminder is active immediately.

Editing and pausing reminders

Tap any schedule in the Schedules list to open it. You can edit any field, or use the Active toggle to pause a reminder without deleting it. This is useful for medications that are only given in a specific course.

Deleting a reminder

Open the schedule and tap the delete icon. The reminder will stop firing immediately. This cannot be undone — create a new one if you need it again.

Notification permissions

Android

VEGA will ask for notification permission on first use. If you declined and want to re-enable it, go to your device Settings → Apps → VEGA → Notifications and turn them on.

For reliable alarm delivery on Android 12+, VEGA also requires the Schedule exact alarms permission. If your reminders seem delayed or don't fire, check Settings → Apps → VEGA → Alarms & reminders and ensure this is allowed.

If your device has aggressive battery optimisation, VEGA may be prevented from waking up for reminders. Go to Settings → Apps → VEGA → Battery → Unrestricted to fix this.

iOS

VEGA will ask for notification permission on first launch. If you declined, go to Settings → Notifications → VEGA and enable Allow Notifications. Make sure Time Sensitive notifications are enabled for the most reliable delivery.

Reminder: If you uninstall and reinstall VEGA, all scheduled reminders need to be recreated. They are stored on-device and are not restored from cloud sync.
